## Technology Stack

### Frontend Layer
- **Gradio 4.16.0**: Interactive web UI framework
- **Python 3.11**: Runtime environment

### Backend Layer
- **FastAPI 0.109.0**: Async web framework
- **Uvicorn**: ASGI server
- **Pydantic 2.5.3**: Data validation
- **SQLAlchemy 2.0.25**: ORM for PostgreSQL

### Data Layer
- **PostgreSQL 16**: Relational database for transactional data
- **Qdrant**: Vector database for semantic search
- **AWS SES**: Email delivery service

### AI/ML Layer
- **Microsoft Presidio 2.2.351**: PII/PHI detection and anonymization
- **spaCy 3.7.2**: NLP engine for Presidio
- **Multi-LLM Support**:
  - OpenAI (GPT-4, GPT-3.5)
  - Azure OpenAI
  - AWS Bedrock (Claude 3)
  - Google Gemini

### Document Processing
- **PyPDF2 3.0.1**: PDF parsing
- **pdfplumber 0.10.3**: Advanced PDF extraction with tables
- **python-docx 1.1.0**: Word document processing
- **openpyxl 3.1.2**: Excel file processing
- **pandas 2.1.4**: Data manipulation

### Infrastructure
- **Docker & Docker Compose**: Containerization
- **nginx (optional)**: Reverse proxy for production

### Performance Optimization

1. **Database**:
   - Connection pooling (10 connections, 20 overflow)
   - Indexed columns for fast queries
   - Prepared statements via SQLAlchemy

2. **Vector Search**:
   - HNSW indexing in Qdrant for fast similarity search
   - Batch embedding generation
   - Collection sharding by user

3. **Caching**:
   - Session caching in memory
   - Query result caching (future enhancement)
   - Static file caching

4. **Async Processing**:
   - FastAPI async endpoints
   - Background task queue (future: Celery)
   - Non-blocking I/O
